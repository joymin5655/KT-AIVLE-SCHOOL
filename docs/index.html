SHEET 03 · AI / ML

자세 분류 파이프라인POSE CLASSIFICATION — TRAIN & INFER

MediaPipe로 뽑은 신체 랜드마크를 거리·각도 피처로 가공해 XGBoost 분류기를 학습시킨다. 학습 산출물 .pkl이 그대로 서버에서 추론에 쓰인다.

DATA데이터 수집 · Data acquisition

video_to_image.py → csv_data_acquisition.py. 폴더명이 곧 라벨 (auto label_dict).

PREP전처리 · Feature engineering

MediaPipe Holistic 33개 랜드마크 → 선택 [0,2,5,7,8,11,12,15,16] (nose·eyes·ears·shoulders·wrists). 좌표 x/y/z/visibility + 랜드마크 쌍 거리 + 눈간거리 정규화 + 삼각 각도(arccos).

MODEL모델링 · Modeling

RandomForest · XGBoost · Deep Learning 비교 → XGBoost 채택. joblib.dump(model, 'pose_classification_model.pkl').

INFER실시간 추론 · Realtime

서버가 프레임마다 동일 전처리 후 pose_model.predict(). 스트레칭 검증은 확장 랜드마크셋 […13,14,21,22](elbows·thumbs) + …_stretch_final.pkl.

0 정상 · good 1–4 불량 유형 · bad-posture types -1 사람 없음 / 저가시성 · no person
